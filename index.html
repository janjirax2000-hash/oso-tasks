ตั้งค่า Google Calendar

ใส่ OAuth Client ID ที่สร้างจาก Google Cloud Console (ประเภท "Web application") โดยต้องเพิ่ม origin ของหน้านี้ (URL https:// ที่ใช้เปิดหน้านี้จริง) ไว้ใน "Authorized JavaScript origins" ค่านี้จะถูกเก็บไว้ในเบราว์เซอร์นี้เท่านั้น หากใช้ร่วมกับทีม สมาชิกแต่ละคนต้องตั้งค่า Client ID เดียวกันในเบราว์เซอร์ของตัวเอง

ตั้งค่าทีม (Firebase)

เชื่อมต่อ Firebase Realtime Database เพื่อให้ทุกคนในทีมเห็นและแก้ไขข้อมูลชุดเดียวกันแบบเรียลไทม์ คัดลอกค่าจาก Firebase Console → Project settings → General → Your apps → SDK setup and configuration ทุกคนในทีมต้องใส่ค่าเดียวกันนี้ในเบราว์เซอร์ของตัวเอง แล้วกด "เชื่อมต่อทีม"

คู่มือการใช้งาน OSO Tasks

📋 ภาพรวม

หน้านี้ใช้บันทึกและติดตามงานประจำของทีม OSO งานทั้งหมดจะถูกจัดกลุ่มแสดงตาม Project เป็นการ์ดแยกกัน คลิกที่หัวการ์ดเพื่อพับ/กางรายการงานในกลุ่มนั้น

➕ เพิ่ม / แก้ไข / ลบงาน

  • เพิ่มงานใหม่: กดปุ่ม "เพิ่มงานใหม่" มุมขวาบน กรอกรายละเอียดแล้วกด "บันทึก"
  • แก้ไขงาน: กดไอคอนดินสอ ✎ ท้ายแถวของงานนั้น
  • ลบงาน: กดไอคอนถังขยะ 🗑 — งานจะไม่หายทันที แต่ถูกย้ายไปเก็บใน "ประวัติที่ลบ" สามารถกด "กู้คืน" กลับมาได้ทุกเมื่อ หรือกด "ลบถาวร" ถ้าไม่ต้องการเก็บแล้วจริงๆ
  • ผู้รับผิดชอบหลายคน: หนึ่งงานมอบหมายได้มากกว่า 1 คน — ในหน้าต่างเพิ่ม/แก้ไขงาน ติ๊กเลือกได้หลายชื่อพร้อมกันในช่อง Responsible แต่ละคนจะโชว์เป็น badge สีในตาราง

🎨 จัดการ Project / ผู้รับผิดชอบ / Status

ปุ่ม "จัดการ Project", "จัดการผู้รับผิดชอบ" และ "จัดการ Status" ในแถบเครื่องมือ ใช้เพิ่ม/แก้ชื่อ/ลบรายการเหล่านี้ได้ โดย Project และผู้รับผิดชอบสามารถเลือกสีเองได้ผ่านช่องสี่เหลี่ยมสีข้างชื่อ (คลิกเพื่อเปิดตัวเลือกสี) ส่วน Status จะได้รับสีให้อัตโนมัติเมื่อเพิ่มใหม่

การลบรายการใดๆ เหล่านี้ งานที่เคยใช้ชื่อนั้นจะยังคงเก็บชื่อเดิมไว้ ไม่หายไปจากงาน เพียงแต่จะไม่โชว์อยู่ในตัวเลือกให้ใช้ต่ออีก

🔍 ค้นหาและกรอง

พิมพ์ในช่องค้นหาด้านบนเพื่อกรองงานตามชื่องาน รายละเอียด หรือหมายเหตุ และคลิกชิป Status ด้านล่าง (เช่น Inprogress, Completed) เพื่อกรองเฉพาะสถานะนั้น

📥 ดาวน์โหลดรายงาน Excel

กดปุ่ม "ดาวน์โหลด Excel" แล้วเลือกได้ว่า:

  • เอางานไหนบ้าง (ติ๊กเลือกเป็นรายตัว หรือค้นหาชื่อก่อนเลือก)
  • เอาหัวข้อคอลัมน์ไหนบ้าง (Status, Detail, Remark ฯลฯ)
  • รูปแบบไฟล์เป็นตาราง (1 แถวต่องาน) หรือรายงานแยกหัวข้อ (1 บล็อกต่องาน คล้ายรายงานสรุป)

💾 สำรอง / นำเข้าข้อมูล

ปุ่ม "สำรองข้อมูล (JSON)" จะดาวน์โหลดข้อมูลทั้งหมดเก็บไว้เป็นไฟล์กันเหตุ ส่วน "นำเข้าข้อมูลสำรอง" ใช้กู้คืนจากไฟล์นั้น (จะแทนที่ข้อมูลปัจจุบันทั้งหมด ควรใช้ด้วยความระมัดระวัง)

👥 โหมดทีม (Firebase)

เมื่อสถานะบนสุดโชว์ "โหมดทีม (Firebase) — เชื่อมต่อแล้ว" หมายความว่าทุกคนที่เปิดลิงก์นี้กำลังดู/แก้ไขข้อมูลชุดเดียวกันแบบเรียลไทม์ — ใครแก้ตรงไหน ทุกคนจะเห็นการเปลี่ยนแปลงทันทีโดยไม่ต้องรีเฟรช

⚠️ เนื่องจากระบบเปิดให้แก้ไขได้ทันทีโดยไม่ต้องล็อกอิน ใครก็ตามที่มีลิงก์นี้สามารถเพิ่ม/แก้ไข/ลบงานได้เหมือนกัน จึงควรส่งลิงก์เฉพาะให้คนในทีมที่ไว้ใจได้เท่านั้น

📅 Google Calendar

งานที่มี "Target Finished" สามารถซิงก์ขึ้น Google Calendar ส่วนตัวได้อัตโนมัติ โดยกดปุ่ม "เชื่อมต่อ" ที่สถานะ Google Calendar มุมบน (ต้องตั้งค่า OAuth Client ID ก่อนใช้งานครั้งแรก) ใช้งานบนมือถือได้เหมือนกันทุกขั้นตอน

งานที่เพิ่ม/แก้ไขหลังเชื่อมต่อแล้วจะขึ้น Calendar ให้อัตโนมัติ ส่วนงานเก่าที่มีอยู่ก่อนเชื่อมต่อ ให้กดปุ่ม "ซิงก์ทั้งหมด" ที่โผล่ขึ้นมาข้างสถานะ Google Calendar (หลังเชื่อมต่อสำเร็จ) เพื่อดันงานเก่าทั้งหมดขึ้น Calendar ทีเดียว

⚠️ ถ้าเปิดลิงก์นี้จากในแอปแชท (LINE, Facebook, Instagram) แล้วกด "เชื่อมต่อ" Google Calendar จะไม่ยอมให้ล็อกอิน — ต้องแตะเมนู "•••" แล้วเลือก "เปิดในเบราว์เซอร์" (Safari/Chrome) ก่อน แอปจะเตือนให้อัตโนมัติถ้าตรวจพบ
